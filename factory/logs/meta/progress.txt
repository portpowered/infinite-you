## Codebase Patterns

- Before treating files under `factory/inputs/**` or `factory/logs/meta/*` as
  checked-in workflow truth, confirm them with `git ls-files`; ignored local
  residue can coexist with the canonical control-plane surface.
- The checked-in meta control plane is `factory/logs/meta/asks.md`,
  `factory/logs/meta/view.md`, and `factory/logs/meta/progress.txt`; any other
  path should be treated as legacy or workspace-local until proven otherwise.
- Before queuing a new meta cleanup lane, compare the candidate file set
  against `gh pr view <number> --json files` for open PRs so the maintainer
  loop does not duplicate active work.
- When the root artifact inventory changes, keep
  `docs/development/root-factory-artifact-contract-inventory.md` and
  `internal/testpath/artifact_contract.go` aligned in the same change or the
  targeted artifact-contract test will fail immediately.
- Before re-queuing a local ignored cleanup idea under `factory/inputs/**`,
  compare it against recent merged PR history; ignored residue can remain in
  the workspace after its lane has already landed on `main`.
- Handwritten routes registered ahead of the generated API router can be
  intentional compatibility shims; verify whether they preserve more tolerant
  request parsing before treating them as removable duplication.
- When a handwritten compatibility route already normalizes tolerant query
  parsing before calling a handler, prefer keeping that behavior in the wrapper
  and removing any second copy of the same fallback logic from the handler.
- A package-local cleanup candidate can still collide with a broad open PR, so
  candidate validation must compare exact touched files against every active
  lane, not just against the candidate's apparent subsystem boundary.
