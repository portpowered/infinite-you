# Checked-In Repository Workflow

This repository ships `./factory/` as a checked-in repository-maintainer
workflow. It is not the default `agent-factory` or `agent-factory init` scaffold,
which still creates the simpler single-step `tasks` workflow documented in the
package README.

The checked-in repository workflow keeps the repository-local automation loop
checked in:

- `thoughts:init` can feed ideation work.
- `idea:init` advances into planning.
- `plan:init` runs `setup-workspace` and creates task work.
- `task:init` and `task:in-review` repeat through `process` and `review`.
- Guarded loop breakers bound repeated retries for both executor and review
  loops.

## Input Layout

Seed checked-in repository work under:

- `inputs/BATCH/default/`
- `inputs/idea/default/`
- `inputs/plan/default/`
- `inputs/task/default/`
- `inputs/thoughts/default/`

Each canonical inbox is kept in git by a tracked `.gitkeep` sentinel so the
directory surface exists in clean checkouts, while the actual work items inside
those folders remain repository-local working state rather than the default
starter generated by `agent-factory init`.

For follow-up work authored by the checked-in maintainer prompts:

- default to one standalone markdown idea file under `inputs/idea/default/`
- use `inputs/BATCH/default/` only when the request needs dependency ordering
  or mixed work types in one canonical `FACTORY_REQUEST_BATCH`

## Replay Samples

`logs/agent-fails.json` is the checked-in event-stream sample used to prove the
event-stream-to-replay conversion path. `logs/agent-fails.replay.json` is the
paired checked-in replay artifact used by the replay smoke.
